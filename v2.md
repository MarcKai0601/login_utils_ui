# Login Utils UI — v2.0 更新說明

> **版本**：v2.0  
> **日期**：2026-03-06  
> **主題**：註冊頁面 + MGR 管理功能 + 明亮/深色模式

---

## 📋 更新摘要

v2 在 v1 SSO 登入頁面的基礎上，新增：
1. **明亮 / 深色模式切換** — 右下角浮動按鈕，持久化至 localStorage
2. **使用者註冊頁面** — 對接後端 `POST /api/mgr/user/add`
3. **MGR 管理儀表板** — 系統狀態總覽 + 快捷操作
4. **個人資料頁面** — 帳號資訊 + JWT Token + RBAC 角色
5. **頂部導航列** — 登入後顯示，可在 Dashboard / Profile 間切換

---

## 🆕 新增頁面

### 1. 註冊頁面 (`/register`)

| 欄位 | 必填 | 說明 |
|------|------|------|
| Username | ✅ | 帳號名稱 |
| Password | ✅ | 最少 6 字元 |
| Confirm Password | ✅ | 確認密碼一致 |
| Nickname | ❌ | 顯示名稱 |
| Email | ❌ | 電子信箱 |
| Phone | ❌ | 電話號碼 |

- 前端驗證：必填檢查、密碼長度、密碼一致性
- 註冊成功後自動導向登入頁
- 紫色 (Violet) 主色調，與登入頁的綠色 (Emerald) 區分

### 2. 管理儀表板 (`/`)

取代原本的 WelcomeView 作為登入後首頁：
- **狀態卡片**：Service 狀態、Auth 機制、RBAC 角色數、Session 資訊
- **快捷操作**：Add New User → 連結至註冊頁、View Profile → 連結至個人資料
- **系統架構**：顯示微服務拓撲（Login Utils UI / Backend / FAS）

### 3. 個人資料頁 (`/profile`)

- User ID、帳號狀態
- JWT Token（截斷顯示）
- RBAC 角色清單（對應 k_user_role + k_role + k_system）

## 🎨 主題系統

| 功能 | 實作方式 |
